# FreeMon — бесплатный мониторинг железа с оверлеем и FPS

Показывает в углу экрана (поверх игр и программ) любые данные о железе:
температуры, нагрузку, частоты, потребление (ватты), напряжения, обороты
вентиляторов, использование памяти, скорость накопителей и сети — и **FPS**.
Полностью бесплатный, открытый, набор показателей выбираете сами.

Аналог MSI Afterburner / RTSS, но без лишнего — только то, что вам нужно.

---

## Как получить готовый `.exe` — БЕЗ установки чего-либо (рекомендую)

Сам `.exe` собирается бесплатно в облаке на серверах GitHub. Вам не нужен
ни компилятор, ни .NET — только браузер и бесплатный аккаунт GitHub.

1. Зарегистрируйтесь на https://github.com (бесплатно).
2. Создайте новый репозиторий: кнопка **New** → имя любое (например `freemon`)
   → **Create repository**.
3. На странице репозитория нажмите **Add file → Upload files**.
   Перетащите туда **всё содержимое этой папки** (включая скрытую папку
   `.github` — она важна, в ней лежит рецепт сборки). Нажмите **Commit changes**.
   - Если папка `.github` не перетаскивается, см. раздел «Если не видно .github» ниже.
4. Откройте вкладку **Actions** вверху репозитория. Сборка запустится сама.
   Подождите 2–4 минуты, пока напротив задачи не появится зелёная галочка.
5. Кликните по завершённой сборке. Внизу страницы, в разделе **Artifacts**,
   будет файл **FreeMon-win-x64** — скачайте его. Внутри лежит `FreeMon.exe`
   и `PresentMon.exe`.
6. Распакуйте, запустите `FreeMon.exe` (Windows спросит права администратора —
   это нужно для чтения датчиков, соглашайтесь).

Готово. `.exe` уже со встроенным .NET — на любом ПК с Windows 10/11 запустится
без доустановок.

> Хотите ещё проще на будущее — создавайте «релизы»: во вкладке репозитория
> **Releases → Create a new release**, тег вида `v1.0.0`. Тогда `.exe` будет
> лежать готовым zip-ом в разделе Releases.

### Если не видно `.github`
Папка `.github` скрытая, и некоторые браузеры её не загружают перетаскиванием.
Тогда создайте её вручную: в репозитории **Add file → Create new file**, в поле
имени введите ровно:
```
.github/workflows/build.yml
```
и вставьте туда содержимое файла `build.yml` из этой папки. Сохраните.

## Альтернатива: собрать у себя на ПК (если не хотите GitHub)

1. Установите **.NET 8 SDK** (бесплатно, от Microsoft):
   https://dotnet.microsoft.com/download/dotnet/8.0
   Нужен именно **SDK**, не Runtime. Установка обычная: далее-далее-готово.
2. Двойной клик по **`build-local.bat`** в этой папке.
3. Через минуту появится `publish\FreeMon.exe`.
4. Для FPS скачайте `PresentMon-...-x64.exe` со
   https://github.com/GameTechDev/PresentMon/releases , переименуйте в
   `PresentMon.exe` и положите рядом с `FreeMon.exe`.

## Как пользоваться

- В большой таблице — все датчики вашего ПК. Поставьте галочку в колонке
  **«Оверлей»** напротив тех, что хотите видеть поверх игр.
- Кнопка **«Показать оверлей»** — включает маленькое окно поверх всего.
  Его можно перетаскивать мышью.
- Когда всё настроили — поставьте галочку **«Зафиксировать (клик сквозь)»**.
  Тогда оверлей перестанет ловить мышь и не будет мешать в игре.
  Чтобы снова подвинуть — снимите эту галочку.
- **Шрифт / Цвет / Фон** — внешний вид оверлея.
- **Интервал, мс** — как часто обновлять цифры (1000 = раз в секунду).
- **«Считать FPS активного окна»** — включает счётчик FPS той игры/программы,
  что сейчас в фокусе. Нужен `PresentMon.exe` рядом (см. выше).

Все настройки и выбранные датчики сохраняются автоматически в
`%AppData%\FreeMon\config.json`.

## Важно про оверлей в играх

- **Работает** для игр в режиме «без рамки» / «оконный без рамки» (borderless),
  а это сегодня большинство игр.
- В режиме **эксклюзивного полного экрана** (старый «fullscreen») поверх-окна
  Windows скрывает — это ограничение самой системы, а не программы. Решение:
  включите в игре режим «без рамки», либо переключайтесь на borderless.
  Полноценный оверлей в эксклюзивном fullscreen умеет только RTSS, который
  внедряется в графику игры (это отдельная большая технология).

## Если что-то не работает

- **Не видно датчиков / мало датчиков** — запускайте от имени администратора
  (программа и так это просит). Без админ-прав драйвер чтения железа не грузится.
- **FPS показывает «—»** — проверьте, что `PresentMon.exe` лежит рядом с
  `FreeMon.exe`, и что игра уже запущена и находится в фокусе.
- **Антивирус ругается** — LibreHardwareMonitor и PresentMon используют
  низкоуровневый доступ к железу/ETW, поэтому антивирусы иногда дают ложное
  срабатывание. Оба компонента открытые и широко используются.

## Что внутри (для интересующихся)

- C# / .NET 8 / WPF.
- `LibreHardwareMonitorLib` — чтение датчиков.
- `PresentMon` (Intel, открытый) — измерение FPS через ETW.
- Лицензии сторонних компонентов — в `THIRD-PARTY.md`.

Структура:
```
FreeMon/
├─ FreeMon.csproj          проект
├─ app.manifest            запрос прав администратора
├─ App.xaml(.cs)           точка входа
├─ MainWindow.xaml(.cs)    главное окно: таблица + настройки
├─ OverlayWindow.xaml(.cs) оверлей поверх игр
├─ Models/                 модели (датчик, настройки, форматирование)
├─ Services/               чтение железа, FPS, сохранение настроек
├─ Interop/                вызовы Windows API (клик-сквозь, активное окно)
└─ .github/workflows/      рецепт облачной сборки .exe
```
